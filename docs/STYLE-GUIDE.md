# 内容风格指南

写每课时严格遵守以下规范。

## 一、字数与密度

| 等级 | 主篇 | 配篇 | 对话 | 词汇 | 语法点 |
|---|---|---|---|---|---|
| N5 | 7-8 句 / 100-150 字 | 6 句 / 80-120 字 | 4-5 轮 | 20-30 | 5-7 |
| N4 | 8-10 句 / 200-300 字 | 6-8 句 / 150-200 字 | 5-6 轮 | 25-35 | 5-7 |
| N3 | 10-12 句 / 300-500 字 | 8-10 句 / 200-300 字 | 6-7 轮 | 30-40 | 5-7 |
| N2 | 12-15 句 / 500-800 字 | 10-12 句 / 300-500 字 | 6-8 轮 | 30-40 | 5-7 |

## 二、文风

### N5
- 主短文用 です・ます 礼貌体
- 句子结构简单，单句不超过两个分句
- 话题：自我介绍 / 房间 / 时间表 / 周末活动 / 喜好
- 视角：第一人称为主

### N4
- 主短文仍以礼貌体为主，可在对话部分加入常体（普通形）
- 句子可以包含 て形连接、过去时、可能形等
- 话题：朋友交流 / 学习 / 旅行 / 愿望 / 比较 / 经验
- 视角：第一人称 + 描述他人

### N3
- 主短文可在专业话题里使用书面体倾向
- 句子开始包含被动、使役、敬语、复杂接续
- 话题：职场 / 社会规则 / 文化习俗 / 新闻评论 / 技术应用
- 视角：可以是观察者、报告者

### N2
- **主短文使用 だ・である 书面体**（关键转变）
- 句子使用复合结构、抽象表达、专业词汇
- 话题：环境 / 教育 / 经济 / 哲学 / 文学 / 历史 / AI 与未来
- 视角：议论文、说明文为主

## 三、ruby 标签使用

**绝对规则**：每个出现的汉字都必须有 `<ruby>...<rt>读音</rt></ruby>` 包裹。

```ts
// ✅ 正确
jp: '<ruby>私<rt>わたし</rt></ruby>は<ruby>学生<rt>がくせい</rt></ruby>です。'

// ❌ 错误：「学生」缺振假名
jp: '<ruby>私<rt>わたし</rt></ruby>は学生です。'

// ❌ 错误：词组应该整体注音，而不是单字注音
jp: '<ruby>学<rt>がく</rt></ruby><ruby>生<rt>せい</rt></ruby>です。'
//   （这样会显示成分裂的两个振假名）
```

**词组注音规则**：
- 二字以上的复合词整体注音：`<ruby>学生<rt>がくせい</rt></ruby>`
- 训读和音读分明的复合词分别注音：`<ruby>朝<rt>あさ</rt></ruby><ruby>ご飯<rt>ごはん</rt></ruby>`
- 名词后的助词（は・を・に等）不需要 ruby

## 四、plain 字段的生成规则

`plain` 是给浏览器 TTS 朗读用的，必须是**所有汉字都替换成假名**的版本。

```ts
{
  jp: '<ruby>毎朝<rt>まいあさ</rt></ruby><ruby>七時<rt>しちじ</rt></ruby>に<ruby>起<rt>お</rt></ruby>きます。',
  plain: 'まいあさしちじにおきます。',  // 注意：所有汉字都替换为假名
  // ...
}
```

**陷阱**：
- 不要保留汉字！否则浏览器 TTS 会读错音（比如「七」可能读成 nana 而不是 shichi）
- 标点保持原样（、。「」）
- 假名和片假名混合时，片假名保持原样（如「コーヒー」不变）

## 五、罗马音规则

使用 Hepburn 式罗马音：

- し=shi、ち=chi、つ=tsu、ふ=fu
- じ=ji、ぢ=ji、ず=zu、づ=zu
- 长音：おう=ou、えい=ei、いい=ii、うう=uu、ああ=aa
- 促音：っ + k/s/t/p → kk/ss/tt/pp
- ん + b/m/p → m（如「さんぽ」=sanpo）
- 复合词中间用空格分隔：「東京駅」=Toukyou-eki

```ts
// 几个常见词的标准罗马音
'おはよう' → 'ohayou'
'こんにちは' → 'konnichiwa'  // は 在此读 wa
'学校' → 'gakkou'
'三時' → 'san-ji'
```

## 六、对话场景设定

每课的 `dialogue.scene` 应：
- 用 1-2 句日文 + 中文描述场景
- 场景与主篇有关联（同一主题、同一情境）
- 自然引入主篇里教过的语法点

`speaker` 字段用：
- 名字（田中、鈴木、山田）—— 优先
- 角色（先生、店員、お客様、母）—— 次选
- 简单代号（A、B）—— 仅在角色不重要时用

## 七、词汇表（vocabulary）

每条 `VocabItem`：

```ts
{
  jp: '電車',           // 含汉字的写法（如有汉字）
  kana: 'でんしゃ',      // 假名读音
  romaji: 'densha',
  meaning: '电车',
  pos: '名詞'           // 词性
}
```

**词性枚举**（types.ts 中已定义，仅可使用以下值）：
- 名詞、動詞I、動詞II、動詞III、い形容詞、な形容詞
- 副詞、助詞、接続詞、指示詞、疑問詞、感動詞、連体詞、慣用句

**重要**：动词分类
- 動詞I（五段动词）：词尾在 う段，如「行く・読む・話す」（ます形：いきます、よみます、はなします）
- 動詞II（一段动词）：词尾「る」前是 い段或 え段元音，如「食べる・見る・寝る」
- 動詞III（不规则）：「来る・する」及其复合词（勉強する、運動する 等）

## 八、语法点（grammar）

每个 `GrammarPoint`：

```ts
{
  pattern: 'N₁ は N₂ より A です',  // 句式，用 N、V、A 等占位符
  name: '比较句',                   // 中文语法名
  explain: '...用法说明...',        // 中文解释
  examples: [
    {
      jp: '...含 ruby 的例句...',
      plain: '...纯假名...',
      cn: '...中文翻译...'
    }
  ]
}
```

**explain 字段写作要求**：
1. 解释用法（什么时候用）
2. 说明变化规则（如果有）
3. 与近义语法的区别（如果存在）
4. 100-200 字之间，一段话讲清楚

❌ 太短：「表示比较。」
✅ 合适：「表示两者比较。结构是 N₁ は N₂ より A です（N₁ 比 N₂ 更 A）。
注意「より」前的是被比较的对象，「は」标记的是主语。否定时用「より...
ません」或「あまり...ません」。与「のほうが」常一起使用，强调一方更...。」

## 九、理解题（questions）

每课 3-5 题，难度梯度：
- 第 1 题：直接事实题（在文中能找到答案）
- 第 2-3 题：需要稍加理解或推断
- 第 4-5 题：需要使用本课语法点回答

每题：
```ts
{
  q: { jp, plain, romaji, cn },  // 问题（4 个字段都要）
  a: { jp, plain, romaji, cn },  // 参考答案
  hint: '中文提示，可选'          // 可选提示
}
```

## 十、目标（goals）

每课的 `goals` 数组：
- 2-3 条
- 中文写
- 形式："能..." 或 "掌握..." 或 "理解..."

例：
```ts
goals: [
  '能用「〜たい」表达自己的愿望',
  '区分「〜たい」（自己愿望）和「〜がほしい」（想要某物）',
  '能描述未来的计划和梦想'
]
```

## 十一、screen 时的设计原则（HTML 渲染）

模板已基于第 2 课实现。改造为数据驱动时保持以下原则：

**视觉**：
- 主色：`#A8895C`（琥珀金）
- 背景：`#F5F1E8`（米白纸张色）
- 文字：`#2A2620`（暖黑）
- 字体：日语用 Noto Serif JP，西文用 IBM Plex Sans
- 大量留白，行间距宽松

**交互**：
- 振假名 / 罗马音 / 中文三层独立开关
- 单击句子朗读单句，全文播放按钮按顺序读完所有句子
- 播放速度：0.7× / 0.85× / 1.0×
- 当前朗读句左侧有琥珀色竖线高亮

**禁止**：
- 不引入任何 CSS 框架（如 Tailwind）
- 不引入任何 JS 库（保持原生）
- 不使用 emoji 装饰
- 不使用渐变背景或鲜艳色彩
